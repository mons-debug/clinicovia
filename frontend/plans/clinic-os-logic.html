Clinic OS — Logic Map

State machines, parallel workflows, every scenario with a clear path. No ambiguity.

1. The Five Entities & Their States

Everything in the clinic maps to these five things. Each has a clear lifecycle. They connect at specific points.

A. Appointment (rendez-vous)

stateDiagram-v2 [*] --> scheduled: Booked (walk-in or planned) scheduled --> confirmed: Patient confirms scheduled --> checked_in: Patient arrives confirmed --> checked_in: Patient arrives checked_in --> in_progress: Doctor starts in_progress --> completed: Doctor finishes scheduled --> no_show: Didn't come (30min+) scheduled --> cancelled: Cancelled confirmed --> cancelled: Cancelled checked_in --> cancelled: Cancelled
Rule: One appointment = one patient + one doctor + one service + one time slot. Appointment OWNS the calendar slot.

B. Patient Journey (queue position)

stateDiagram-v2 [*] --> awaiting: Arrives at clinic [*] --> in_room: Fast-track (clinic empty) awaiting --> in_room: Called by doctor in_room --> checkout: Doctor finishes (Terminer) checkout --> done: Paid + released awaiting --> left: Patient leaves before seen
Rule: Queue = today only. Resets every morning. Stale entries from yesterday get flagged for cleanup.

C. Facture (invoice)

stateDiagram-v2 [*] --> draft: Doctor starts séance draft --> draft: Reception edits draft --> issued: Reception validates issued --> partial: Partial payment issued --> paid: Full payment partial --> paid: Remaining paid draft --> cancelled: Cancelled issued --> cancelled: Cancelled
Rule: Facture stays DRAFT until reception validates. No payment possible on DRAFT. Reception MUST review before issuing.

D. Consent

stateDiagram-v2 [*] --> pending: Created on séance start pending --> signed: Patient signs at reception pending --> declined: Patient refuses declined --> [*]: Séance blocked
Rule: If declined → séance cannot proceed. Doctor is notified. Appointment → cancelled. DRAFT facture → cancelled.

E. Plan & Séances

stateDiagram-v2 state "Plan" as plan { [*] --> active: Created active --> completed: All sessions done active --> abandoned: Patient stops mid-way } state "Séance" as seance { [*] --> scheduled_s: Auto-created with plan scheduled_s --> in_progress_s: Doctor starts in_progress_s --> completed_s: Doctor finishes scheduled_s --> cancelled_s: Cancelled }
Rule: Each séance = one appointment in calendar. Plan abandoned → remaining séances cancelled → their appointments cancelled → their DRAFT factures cancelled.

3. The Parallel Workflow

This is the key insight: doctor and reception work in parallel, not sequentially. While doctor treats the patient, reception handles documents.

🩺 Doctor (en consultation)

  1. Opens séance → "Commencer"
  2. Edits consent text + price if needed
  3. Sends to reception
  4. — treats patient —
  5. Takes before photos
  6. Writes SOAP note
  7. Takes after photos
  8. Creates ordonnance (optional)
  9. Clicks "Terminer"

📋 Reception (in parallel)

  1. Sees "Documents reçus" notification
  2. Prints consent → patient signs
  3. Marks consent as signed
  4. Reviews facture (DRAFT)
  5. Edits if needed (items, price, discount)
  6. Validates facture → ISSUED
  7. — waits for doctor to finish —
  8. Patient moves to "À encaisser"
  9. Collects payment → prints facture + ordonnance
  10. Releases patient

Critical Timing Rule

What if doctor clicks "Terminer" but reception hasn't validated the facture yet?

Two options:

Option A: BlockOption B: Allow (recommended)
Doctor can't click Terminer until facture is ISSUED. Forces reception to act first. Doctor clicks Terminer freely. Patient moves to "À encaisser" with DRAFT facture. Reception validates + collects payment there. More flexible — doctor isn't blocked by reception speed.
Decision: Option B. Doctor and reception work independently. If facture is still DRAFT when patient reaches "À encaisser", reception validates it there. No blocking.

4. Every Scenario — Clear Path

S1: Normal appointment (happy path)

Book → Arrive → Salle d'attente → Doctor calls → En consultation → Documents sent → Reception signs consent + validates facture → Doctor finishes → À encaisser → Pay → Print → Release
EntityPath
Appointmentscheduled → checked_in → in_progress → completed
Queueawaiting → in_room → checkout → done
Consentpending → signed
Facturedraft → issued → paid

S2: Walk-in, clinic empty (fast-track)

Walk-in → Pick doctor + service → Appointment created → Skip salle d'attente → Directly en consultation → normal flow continues
EntityPath
Appointmentscheduled → in_progress (skip checked_in)
Queuedirectly in_room (skip awaiting)
Trigger: Reception sees doctor has no patient in_room and no one awaiting. "Envoyer directement" button appears on walk-in dialog.

S3: Walk-in, calendar full

Walk-in → Pick doctor → Calendar shows "complet aujourd'hui" → Shows next available slot per doctor → Book for another day OR add to end of today (doctor decides)

Reception sees: "Dr. Meryem: prochaine dispo demain 09h30" / "Dr. Amr: prochaine dispo aujourd'hui 16h00"

No waitlist needed. Just show next available and let reception book it. Simple.

S4: Patient no-show

Appointment time passes → 15min grace → Reception clicks "Absent" → Slot freed → No-show count +1 on dossier
EntityPath
Appointmentscheduled → no_show
Queuenever entered
Calendarslot freed immediately
Patient dossierno_show_count += 1
No auto-mark. Reception decides — maybe patient called to say they're late. Only manual "Absent" button.

S5: Consent refused

Consent printed → Patient reads → Refuses to sign → Reception marks "Refusé" → Doctor notified → Séance cancelled → DRAFT facture cancelled → Patient released or rescheduled
EntityPath
Consentpending → declined
Séancein_progress → cancelled
Facturedraft → cancelled
Appointmentin_progress → cancelled
Queuein_room → checkout (to release) or awaiting (to reschedule)
Cascade: consent declined triggers automatic cancellation of séance + draft facture. Doctor sees "Consentement refusé" alert in their dossier view.

S6: Patient cancels mid-plan

Plan has 6 sessions, 3 completed → Patient says stop → Reception/doctor marks plan "Abandonné" → Remaining sessions + appointments cancelled
WhatAction
Completed sessions (1-3)Untouched. History preserved.
Paid factures (1-3)Untouched. Already paid.
Remaining sessions (4-6)→ cancelled
Their appointments→ cancelled (calendar freed)
Their DRAFT factures (if any)→ cancelled
Plan statusactive → abandoned

S7: Patient arrives early, wants earlier slot

Patient has 15h appointment → Arrives at 13h → Doctor has gap at 13h30 → Reception clicks "Avancer" → Appointment moves to 13h30 → Patient enters queue
Reception sees doctor's live calendar. If a gap exists, "Avancer le RDV" button appears. Appointment time updates, calendar updates, queue order updates.

S8: Patient wants additional service

Patient came for Botox (Meryem) → Also wants Rhinoplastie consultation (Amr) → Reception books second appointment for another day
WhatLogic
Same patient, same day, same doctor?Possible but rare. Add extra time to current appointment.
Same patient, same day, different doctor?Only if patient finishes first appointment. Sequential, not parallel.
Same patient, different day?Normal booking. Just a new appointment linked to same dossier.
All appointments — same dossier. Patient dossier shows every appointment across all doctors, all services, all dates. One patient = one dossier = complete history.

S9: Appointment runs over

Botox (30min) started at 10h00 → It's 10h40, still in_progress → Calendar shows overrun in red → Next patient's wait time updated
WhatLogic
CalendarSlot visually extends past scheduled end. Red border = overrun.
Next patientQueue shows updated estimated wait time based on actual consultation duration.
No auto-actionDoctor finishes when they finish. System just reflects reality.
Calendar = reality. When appointment is in_progress, actual start time is tracked. Duration = max(service_duration, actual_duration). Calendar shows actual, not planned.

S10: Split payment

Facture: 1500 MAD → Patient pays 1000 cash → Receipt printed → Facture: PARTIAL (500 remaining) → Next visit: pays 500 by card → Facture: PAID
Payment dialog at caisse: enter amount + pick method (cash/card/cheque/virement). Can record multiple payments. Facture stays PARTIAL until total covered. Outstanding balance visible on patient dossier.

S11: Doctor absent

Doctor marks "Absent aujourd'hui" → All their scheduled appointments for today flagged → Reception sees alert → Bulk reschedule or cancel
Not auto-cancel. Reception decides per appointment: reschedule (pick new date) or cancel + notify patient. Calendar shows doctor's column as blocked.

S12: End of day cleanup

20h00 → Queue has 0 patients (normal) OR has stale entries → Next morning: system flags yesterday's unresolved entries
Morning check: Any patient still "awaiting" or "in_room" from yesterday? Flag for reception to clean up. Don't auto-resolve — could be data entry that was missed.

S13: Patient wants to modify existing appointment

Patient calls → Change date, service, or doctor
ChangeLogic
Change date/timeShow available slots for same doctor + service. Move appointment.
Change serviceDuration may change → available slots may differ. Update appointment.
Change doctorDifferent calendar entirely. Cancel old, create new with new doctor's service.
