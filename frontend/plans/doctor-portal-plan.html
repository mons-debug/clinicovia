🚨 Reception Workflow — Current vs. Fixed

Current Problem

❌ Current (broken)✅ Fixed
Doctor starts séance → facture created as ISSUED immediately Doctor starts séance → facture created as DRAFT
Reception sees "Payé" button — can't edit amount, items, or discount Reception sees "Réviser" button — can edit items, adjust price, add discount, then validate
Consent + payment mixed in "En consultation" column Clear separation: consent signing during consultation, payment at checkout
"À encaisser" column only shows ordonnances "À encaisser" shows: validated facture (ready to pay) + ordonnances + final "Payé" + "Libérer"

Fixed Flow — Step by Step

1. Doctor starts séance
👨‍⚕️ Doctor portal
  • Clicks "Commencer"
  • Edits consent + price if needed
  • Sends to reception
2. Reception: documents
📋 En consultation column
  • Sees "Documents reçus"
  • Consent: imprimer → signer
  • Facture (DRAFT): réviser → éditer → valider
3. Doctor finishes
👨‍⚕️ Doctor portal
  • Completes séance steps
  • Clicks "Terminer la visite"
  • Creates ordonnance if needed
  • Patient → À encaisser
4. Caisse: payment
💰 À encaisser column
  • Facture is ISSUED (validated)
  • Reception sees total
  • Collects payment (cash/card/cheque)
  • Print facture PDF
5. Release
📋 Reception
  • Print ordonnances
  • Give documents to patient
  • Click "Libérer"
  • Patient done ✓

The Big Picture

Core principle: Doctors own their services. Everything else (plans, séances, calendar, walk-in, factures, consents) derives from the doctor + service pair. Reception validates documents before payment.

Connected Flows

🚶 Walk-in (Reception)

  1. Patient arrives → Walk-in dialog
  2. Fill/select patient info
  3. Pick doctor (Amr / Meryem)
  4. Pick service (filtered by doctor)
  5. → Appointment created (doctor + service + duration)
  6. → Patient enters queue assigned to doctor

📋 Create Treatment Plan

  1. From patient dossier → "Nouveau plan"
  2. Pick doctor → pick service
  3. Number of sessions + interval
  4. Price auto-fills from service config
  5. → Sessions auto-created, linked to doctor+service
  6. → Appointments auto-scheduled in doctor's calendar

▶️ Commencer Séance

  1. Consent pre-fills from doctor's service template
  2. Price pre-fills from doctor's service config
  3. Doctor edits if needed ✓
  4. Consent created (PENDING)
  5. Facture created as DRAFT
  6. → Sent to reception for review

📅 Calendar (restructured)

  1. Day view: columns per doctor
  2. Slots sized by service duration
  3. Color-coded by service category
  4. New appointment: pick doctor → service → slot
  5. No double-booking (exact durations)
  6. Walk-in appointments appear live

Build Phases (8 phases)

1
Doctor Services — Model + API + Migration
Create doctor_services table + CRUD API. Seed Refine services for Amr + Meryem. Add doctor_service_id FK to appointments + treatment_plans. Migrate existing data.
2
Doctor Portal — "Mes Services" Page
New page: list/add/edit/remove services. Configure consent template + price + duration. Sidebar link for doctor role only.
3
Walk-in + Appointment → Doctor + Service Picker
Refactor WalkInDialog: pick doctor → pick service. Same for new appointment. Salle d'attente shows assigned doctor on patient card.
4
Reception Workflow Fix CRITICAL
Facture: DRAFT → review → ISSUED → pay.
• prepare-session creates facture as DRAFT (not ISSUED)
• "En consultation" column: reception can preview + edit facture (items, price, discount) + validate
• "À encaisser" column: only ISSUED factures → collect payment (cash/card/cheque) + print PDF
• Clear step: validate facture → then accept payment. No mixing.
5
Plans & Séances → Linked to Doctor Service
NewPlanDialog: doctor + service picker → price/duration auto-fill. Sessions inherit doctor_service. Commencer séance pre-fills from config.
6
Calendar Restructure — Doctor Columns
Day view: one column per doctor. Slots by duration. Color by category. New appointment flow: pick doctor → service → slot. Prevents double-booking.
7
Professional Documents (PDFs)
New consent PDF template. Logo in all templates. Consent PDF endpoint. Global consent (first visit). Logo upload in settings.
8
Dossier Integration + Polish
Patient dossier: consents with PDF, factures section, global consent badge. Wizard: consent indicator. Queue: PDF download buttons.
